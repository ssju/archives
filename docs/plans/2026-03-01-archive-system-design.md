# 档案管理系统架构设计文档

**文档版本：** v1.0
**创建日期：** 2026-03-01
**设计方案：** 经典分层架构

---

## 一、项目概述

### 1.1 项目背景
开发一个功能完善的档案管理系统，支持文件录入、档案整编、OCR识别、AI智能检索等核心功能，满足2000用户、100万档案量的中型系统需求。

### 1.2 非功能性需求
- **系统规模：** 2000总用户，80-120并发用户，100万档案
- **性能要求：** 页面加载1秒内，查询500ms内，OCR实时进度反馈
- **OCR/AI方案：** 云服务（阿里云/腾讯云OCR + 通义千问API）
- **数据安全：** 每日备份，单副本存储，基础RBAC权限控制
- **可用性：** 准7x24运行，99%可用性
- **终端支持：** PC + 移动端响应式设计
- **扫描仪对接：** 后期扩展（第一期不实现）

### 1.3 技术栈
- **前端：** Vue3 + Element Plus + Vant
- **后端：** Java SpringBoot JDK 1.8
- **数据库：** MySQL 8.0 + Redis 6.x + Elasticsearch 7.x
- **部署：** Docker + Docker Compose

```
│  ┌──────────┬──────────┬──────────┬──────────┐          │
│  │文件管理  │档案整编  │智能检索  │权限管理  │          │

│  └──────────┴──────────────────────────────┘            │
└─────────────────────────────────────────────────────────┘
```

### 2.2 核心设计理念
- 前后端分离，接口RESTful化
- 模块化设计，高内聚低耦合
- 异步处理耗时任务（OCR、AI）
- 多级缓存提升查询性能
- 容器化部署，易于扩展

## 四、核心模块设计

### 4.1 文件管理模块 (file-management)
**职责：**
- 文件上传/下载/删除/移动
- 文件列表查询（分页、筛选、排序）
- 文件详情查看
- 文件预览（图片、PDF、Office文档）
- 文件导出（单个/批量压缩包，加密）
- 支持已归档/未归档文件分类管理

### 4.2 档案整编模块 (archive-editing)
**职责：**
- 文件拖拽排序
- 多图片合并为PDF
- 案卷合并/拆分
- 智能识别漏页/多页/空白页/错序
- 自动调整修复
- 缩略图生成

### 4.3 OCR处理模块 (ocr-service)
**职责：**
- 对接阿里云OCR API
- 异步任务队列（RabbitMQ）
- OCR进度实时推送（WebSocket）
- 识别结果存储和管理
- 原件/识别件对比查阅
- 识别内容修改和保存

### 4.4 智能检索模块 (search-service)
**职责：**
- 关键词全文检索（Elasticsearch）
- 元数据筛选查询
- 语义化智能检索（通义千问API）
- 检索结果高亮显示
- 检索历史记录
- 相关文档推荐

### 4.5 AI服务模块 (ai-service)
**职责：**
- 档案摘要自动生成（通义千问API）
- 关键信息提取
- 文档关联分析
- 智能问答
- 线索挖掘

### 4.6 统计分析模块 (statistics-service)
**职责：**
- 多维度数据统计（档案类型、所属单位、时间、进度等）
- 图表可视化（ECharts）
- 调用排行榜
- 编目/校验比例统计
- 数据导出

### 4.7 权限管理模块 (permission-service)
**职责：**
- 用户管理（增删改查）
- 角色管理（录入员、管理员、系统管理员、普通用户）
- 权限分配（基于RBAC）
- 三权分立管控（系统管理权、安全审计权、安全管理权）
- 登录认证（JWT）
- 操作日志审计

### 4.8 系统配置模块 (config-service)
**职责：**
- 文件尺寸限制配置
- 大模型API配置
- 档案元属性配置（自定义字段）
- 案卷层级配置
- 系统参数管理

### 4.9 模块间通信
- **同步调用：** 直接方法调用（单体应用内）
- **异步通信：** RabbitMQ消息队列（OCR任务、AI任务）
- **实时通信：** WebSocket（进度推送）

## 五、数据库设计

### 5.1 MySQL 核心数据表

**档案相关表：**
- `archive_file` - 档案文件主表（档号、标题、分类、层级、状态等）
- `archive_metadata` - 档案元数据表（动态字段，支持自定义）
- `archive_case` - 案卷表（案卷信息、层级关系）
- `archive_case_relation` - 案卷文件关联表（案卷与文件的多对多关系）
- `archive_page` - 档案页表（图片文件的每一页）

**OCR和AI相关表：**
- `ocr_task` - OCR任务表（任务状态、进度、结果）
- `ocr_result` - OCR识别结果表（原文、识别文本、置信度）
- `ai_summary` - AI摘要表（档案摘要、关键信息）
- `ai_relation` - 档案关联关系表（智能关联的档案）

**系统管理表：**
- `sys_user` - 用户表
- `sys_role` - 角色表
- `sys_permission` - 权限表
- `sys_user_role` - 用户角色关联表
- `sys_role_permission` - 角色权限关联表
- `sys_config` - 系统配置表
- `sys_metadata_config` - 元数据配置表（自定义字段定义）
- `sys_operation_log` - 操作日志表（审计）

**业务辅助表：**
- `file_upload_record` - 文件上传记录表
- `search_history` - 检索历史表
- `export_task` - 导出任务表

### 5.2 索引策略
- **主键索引：** 所有表的 `id` 字段
- **唯一索引：** 档号、用户名等
- **普通索引：** 档案分类、状态、创建时间、所属单位等高频查询字段
- **联合索引：** (user_id, create_time)、(archive_type, status) 等组合查询场景
- **全文索引：** 档案标题（MySQL 8.0支持中文全文索引，作为ES的补充）

### 5.3 Redis 使用方案
- **用户会话：** `session:{token}` (30分钟过期)
- **热点数据缓存：** `archive:detail:{id}` (1小时过期)
- **查询结果缓存：** `search:result:{hash}` (10分钟过期)
- **OCR任务进度：** `ocr:progress:{taskId}` (实时更新)
- **分布式锁：** `lock:archive:{id}` (防止并发编辑)
- **限流计数：** `ratelimit:api:{userId}` (API调用频率限制)

### 5.4 Elasticsearch 索引设计
- **archive_index** - 档案全文索引
  - 字段：档号、标题、OCR识别文本、元数据、摘要
  - 分词器：IK中文分词器
  - 支持高亮、模糊匹配、权重排序
- **search_log_index** - 检索日志索引（用于统计分析）

### 5.5 数据一致性保证
- **MySQL 与 ES 数据同步：** 通过 RabbitMQ 异步同步，保证最终一致性
- **Redis 缓存更新：** 写操作时删除缓存，读操作时重建缓存
- **分布式事务：** 使用本地事务 + 补偿机制（不使用分布式事务框架）

## 六、部署架构

### 6.1 Docker 容器编排

```yaml
# docker-compose.yml 结构
services:
  nginx:          # 反向代理 + 负载均衡
  app-1:          # SpringBoot 应用实例1
  app-2:          # SpringBoot 应用实例2
  mysql:          # MySQL 8.0 数据库
  redis:          # Redis 缓存
  elasticsearch:  # Elasticsearch 搜索引擎
  rabbitmq:       # RabbitMQ 消息队列
  minio:          # MinIO 对象存储（可选）
  prometheus:     # 监控数据采集
  grafana:        # 监控可视化
```

### 6.2 容器配置要点

**Nginx 容器：**
- 端口映射：80 (HTTP) / 443 (HTTPS)
- 配置负载均衡到 app-1 和 app-2
- 静态资源缓存
- WebSocket 代理配置（OCR进度推送）
- SSL 证书挂载

**SpringBoot 应用容器（2个实例）：**
- 基础镜像：openjdk:8-jdk-alpine
- 内存限制：2GB per instance
- 环境变量：数据库连接、Redis地址、ES地址等
- 健康检查：/actuator/health
- 日志挂载：./logs:/app/logs

**MySQL 容器：**
- 版本：mysql:8.0
- 端口：3306
- 数据持久化：./mysql-data:/var/lib/mysql
- 配置文件挂载：./mysql.cnf
- 定时备份脚本：每日凌晨2点自动备份

**Redis 容器：**
- 版本：redis:6.2-alpine
- 端口：6379
- 持久化：AOF + RDB
- 最大内存：2GB，淘汰策略：allkeys-lru

**Elasticsearch 容器：**
- 版本：elasticsearch:7.17.0
- 端口：9200, 9300
- 内存：JVM heap 2GB
- IK 分词器插件
- 数据持久化：./es-data:/usr/share/elasticsearch/data

**RabbitMQ 容器：**
- 端口：5672 (AMQP), 15672 (管理界面)
- 队列：ocr_queue, ai_queue
- 持久化消息

### 6.3 监控告警方案
- **Prometheus** 采集指标（应用、数据库、Redis、ES）
- **Grafana** 可视化仪表盘
- **告警规则：** CPU > 80%、内存 > 85%、磁盘 > 90%、接口响应时间 > 2s
- **告警通知：** 邮件/钉钉/企业微信

### 6.4 部署流程
1. 服务器准备（推荐配置：8核16GB，500GB SSD）
2. 安装 Docker 和 Docker Compose
3. 上传项目文件和配置
4. 执行 `docker-compose up -d` 启动所有服务
5. 初始化数据库（执行 SQL 脚本）
6. 访问系统并验证功能

### 6.5 扩展性考虑
- **应用层：** 可通过增加 app 实例数量水平扩展
- **数据库：** 预留主从架构升级空间
- **文件存储：** 可从 MinIO 迁移到阿里云 OSS
- **缓存：** 可升级为 Redis Cluster

## 七、关键技术方案

### 7.1 OCR异步处理和进度推送

**流程设计：**
```
用户上传图片 → 创建OCR任务 → 发送到RabbitMQ队列
                                    ↓
                            后台消费者处理
                                    ↓
                    调用阿里云OCR API识别
                                    ↓
                    保存识别结果到数据库
                                    ↓
                    更新Redis进度缓存
                                    ↓
            通过WebSocket推送进度到前端
```

**技术实现：**
- **RabbitMQ 队列：** `ocr_queue`，支持任务优先级
- **消费者：** 3个并发消费者，提高处理速度
- **WebSocket：** Spring WebSocket，建立长连接推送进度
- **进度状态：** 待处理(0%) → 识别中(1-99%) → 已完成(100%) → 失败(-1)
- **失败重试：** 最多重试3次，失败后记录日志并通知用户

### 7.2 AI大模型集成方案

**摘要生成：**
- **触发时机：** OCR识别完成后自动触发
- **处理方式：** 异步队列 `ai_queue`
- **API调用：** 通义千问 API，提示词模板化
- **提示词示例：**
  ```
  请为以下档案内容生成200字以内的摘要，提取关键信息：
  档案标题：{title}
  档案内容：{content}
  ```
- **结果存储：** 保存到 `ai_summary` 表，关联档案ID

**智能检索：**
- **检索方式：** 关键词检索 + 语义检索混合
- **关键词检索：** Elasticsearch 全文检索（快速）
- **语义检索：** 通义千问 API 理解用户意图，返回相关档案
- **结果融合：** 关键词匹配结果 + 语义相关结果，按相关度排序
- **检索理由：** AI生成每个结果的匹配理由，提升用户体验

### 7.3 文件存储方案

**存储策略：**
- **原始文件：** 阿里云OSS（或MinIO）
- **缩略图：** 本地存储（Nginx直接访问，提升速度）
- **临时文件：** 本地临时目录，定时清理

**文件组织结构：**
```
/archives/
  /{year}/          # 按年份分目录
    /{month}/       # 按月份分目录
      /{archive_id}/  # 按档案ID分目录
        /original/    # 原始文件
        /ocr/         # OCR识别文件
        /thumbnail/   # 缩略图
```

**文件访问：**
- **上传：** 前端 → 后端 → OSS，返回文件URL
- **下载：** 前端 → 后端生成临时访问URL（有效期1小时）→ 前端直接从OSS下载
- **预览：** 前端直接访问OSS临时URL
- **导出：** 后端从OSS下载 → 打包压缩 → 加密 → 返回下载链接

### 7.4 权限管理（三权分立）方案

**角色设计：**
- **系统管理员：** 系统配置、用户管理、角色管理
- **安全审计员：** 查看操作日志、审计报告、不能修改数据
- **安全管理员：** 权限分配、密码策略、不能访问业务数据
- **业务管理员：** 档案管理、文件整编、统计分析
- **录入员：** 文件录入、基础编辑
- **普通用户：** 查询、浏览、下载

**权限控制：**
- **认证：** JWT Token，有效期2小时，支持刷新
- **授权：** 基于RBAC（角色-权限-资源）
- **数据权限：** 支持按部门、按档案类型的数据隔离
- **操作审计：** 记录所有增删改操作，包含操作人、时间、IP、操作内容

**三权分立实现：**
- 系统管理权、安全审计权、安全管理权互相独立
- 任何敏感操作需要双人审批（如删除档案、导出数据）
- 关键配置修改需要安全管理员授权

### 7.5 性能优化方案

**查询优化：**
- 数据库索引优化
- Redis多级缓存（热点数据、查询结果）
- Elasticsearch分页优化（使用scroll API处理大结果集）

**接口优化：**
- 列表接口：分页查询，默认20条/页
- 详情接口：Redis缓存，1小时过期
- 统计接口：定时任务预计算，存入Redis

**前端优化：**
- 图片懒加载
- 虚拟滚动（大列表）
- 路由懒加载
- CDN加速静态资源

**并发控制：**
- Nginx限流：单IP每秒最多100请求
- 应用层限流：Redis + 令牌桶算法
- 数据库连接池：Druid，最大连接数50

## 九、风险评估

### 9.1 技术风险
- **OCR识别准确率：** 依赖阿里云OCR服务质量，建议前期充分测试
- **AI API成本：** 大模型调用成本可能较高，需要控制调用频率
- **性能瓶颈：** 100万档案量下的查询性能需要持续优化

### 9.2 缓解措施
- OCR识别支持人工校正
- AI调用增加缓存机制，避免重复调用
- 数据库分库分表预案（如数据量超预期）

本设计方案采用经典分层架构，技术栈成熟稳定，适合2000用户、100万档案的中型系统需求。通过模块化设计、异步处理、多级缓存等手段，可以满足快速级性能要求。Docker化部署简化了运维复杂度，预留了微服务拆分的扩展空间。
