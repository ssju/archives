# 智能档案管理系统 - 产品需求文档 (PRD V4.0)

本项目旨在打造一个结合现代 AI 技术（OCR、大模型、知识图谱）与专业档案管理标准（依据 DA/T 13-2022《档号编制规则》与 DA/T 18-2022《档案著录规则》）的新一代**智能档案管理系统**。本版本在前版页面流转和核心架构基础上，针对**数据导入校验流**、各界面的**详细功能点配置**及**操作反馈细节**进行了全面丰富的扩充，并详细定义了**文件管理列表字段和档号编制规范**。

---

## 页面与系统模块详细设计说明

### 一、 系统登录与权限路由页
**1. 页面定位**：系统的第一道安全防线。
**2. 界面清单与功能细节**：
*   **登录方式**：支持账号密码登录、支持企业微信/钉钉扫码单点登录，支持具备高安全要求的硬件（USB Key/指纹）接入。
*   **验证机制**：内置滑块验证或动态行为验证码，防暴力破解。
*   **动态路由**：登录成功后，系统根据用户拥有的**角色与所属部门**动态分配可见的左侧导航菜单，并计算好可以查阅的数据权限范围，将用户引导至相应的首页大屏。

### 二、 首页大屏 (系统门户与数据看板)
**1. 页面定位**：登录后的主要着陆页，提供宏观的档案资产盘点与个人的快捷工作入口。
**2. 界面清单与功能细节**：
*   **顶部统计总览区**：馆藏总档案数、本月新增档案数、累计检索次数、累计借阅文件总数（数字卡片动态跳动展示）。
*   **动态图表区 (深度下钻支撑)**：
    *   **档案类型/部门分布（饼图/环形图）**：展示各类档案（文书、会计、项目等）的数量占比。点击饼图扇区，可直接跳转并在【文件管理】中带上对应的档案分类筛选条件。
    *   **业务处理漏斗（柱状图/折线图）**：呈现每日导入量 -> 智能编目量 -> 审核归档量 -> 借阅利用量的转化趋势。
    *   **知识热力引擎（词云）**：基于用户的高频检索词及 AI 提取的关键高频实体，形成视觉词云，点击热词跳转全库检索。
*   **个人工作台 (中下部卡片视图)**：
    *   **待办事项提醒**：包含【待我审批的借阅申请】（高亮红色角标）、【我发起的借阅-已被驳回/已通过】。
    *   **快捷工作通道**：提供“最近查阅文档（历史区）”、“我的收藏夹”、“快速上传入口”的直达按钮。

### 三、 文件录入与管理中心 (核心工作台及著录规范)
**1. 页面定位**：档案管理人员的核心操作台，遵循 DA/T 18-2022 著录要求，保障数据标准化。
**2. 界面清单与功能细节**：
*   **视窗布局**：左侧为【全宗及分类目录树】（支持无限极分类，支持右键重命名、新建分类）。右侧顶部为高级筛选栏，中间为【数据列表展示区】，底部为分页组件。系统支持**已归档视图**与**预归档（散件）视图**切换。
*   **【重点】文件管理列表字段定义 (严格依据 DA/T 18-2022 著录规则)**：
    为满足专业化要求，列表支持横向滚动，并默认展示以下核心控制字段与标识字段，用户亦可自定义展示隐藏列：
    *   **核心字段区 (冻结列首)**：
        1.  **✅ 档号 (Reference Code)**：唯一标识，支持点击复制。
        2.  **✅ 题名 (Title)**：档案的法定名称/标题，长度自适应。
    *   **背景与内容项区**：
        3.  **责任者 (Creator)**：文件的制定/发文单位或个人。
        4.  **文件编号/发文字号 (Document Number)**：如“国办发〔2022〕1号”。
        5.  **形成时间/归档日期 (Date)**：如“2022-07-01”。
        6.  **全宗名称及全宗号 (Fonds Name/Number)**：档案所属的最高实体基本单位。
    *   **访问与利用控制项区**：
        7.  **密级 (Security Classification)**：[公开 / 内部 / 秘密 / 机密 / 绝密]（高亮不同颜色）。
        8.  **保管期限 (Retention Period)**：[10年 / 30年 / 永久]。
    *   **系统扩展辅助区**：
        9.  **挂接状态**：[已挂接电子件 / 无电子件]。
        10. **系统标签**：标签库挂载回显列。

*   **安全强校验导入流 (脏数据阻断引擎)**：
    *   **第一步：导入 Excel 编目目录 -> 系统级强校验拦截**
        系统读取 Excel 后，立刻与后台的**档案元数据规则字典**进行碰撞比对。
        **阻断性错误拦截**：必填项为空（如缺少密级、保管期限）、枚举值越界（如系统设定的密级只有“公开/机密”，但 Excel 里填了“绝密”）、日期格式非法。系统直接抛红，**禁止入库，并提供异常明细下载**。必须修正后重启导入。
    *   **第二步：导入实体电子文件 -> 自动挂接与双向诊断**
        系统将物理拖拽的文件与第一步导入的目录数据进行匹配。生成**匹配对账单**：“有档无文件（缺少电子件）”、“有文件无档（属于散乱文件，拦截至预归档散件区）”、“完全匹配成功”。
*   **列表页轻量级悬浮交互**：
    鼠标悬浮于文件题名上，旁边浮现半透明气泡框，显示：【缩略首图】、【文件体积与页数】、【AI 生成摘要】、【核心标签】，不打断用户的连续翻页审核动作。

### 四、 档案整编工作台 (数字化加工流水线)
**1. 页面定位**：针对散件或扫描件进行二次装订、物理修复的沉浸式加工车间。
**2. 界面清单与功能细节**：
*   **功能布局**：左侧展示案卷节点结构，右侧展示多图画板（瀑布流缩略图），支持跨卷拖拽移动、拆卷、重组。
*   **CV 智能辅助质检警报**：
    系统在后台对图像进行静默分析。检测到歪斜、大面积黑边、明显空白页时，在缩略图打上“红色叹号”，提醒编目员注意，并提供“一键自动纠偏裁边”等修复按钮。
*   **高级卷操作（拆卷/合卷）**：
    支持一键拆卷为单件，也支持将多个图片文件一键合并并输出带有书签的标准版式文件(PDF/OFD)。在合卷时，支持依据识别出的页码或自然段落语义，由 AI 给出建议的最佳合并顺序。

### 五、 文件详情阅读页 (深度查阅与内容校对)
**1. 页面定位**：用于审核员精细化审阅文件内容、校对识别错误并补充元数据的终端界面。
**2. 界面清单与功能细节**：
*   **左侧：可折叠卷宗导航**：展示同卷其他文件目录列表。
*   **中区：双页并排沉浸式对比视窗 (图片处理核心场景)**：
    **左半边渲染超清原图，右半边显示OCR文本框**。光标点左侧某一行，右侧自动对齐跟随。修改右侧错误文字即发触发保存。
*   **右区：抽屉式元属性与智能洞察面板**：
    *   在元数据栏：展示系统所有的著录项。提供**“一键 AI 提取”按钮，系统自动阅读正文并帮用户精准提取“题名”、“责任者”、“文号”、“时间”等结构化细项，极大节省手工著录时间**。
    *   PDF页内支持高级关键字检索映射高亮锚点区。

### 六、 全域智能检索与信息利用
**1. 页面定位**：获取档案价值的统一门户，借阅发起的职能端。
**2. 界面清单与功能细节**：
*   **组合高级检索**：支持关键词全文匹配 + DA/T 18 著录属性多维叠加检索 + **语义化自然语言问答检索**。
*   **搜索结果混合分级展示 (数据脱敏策略)**：
    *   越权档案（仅能看属于别人的本级或上级机密档案）：标题脱敏处理（***替代），摘要提取非机密中性词。
    *   右侧提供【申请查阅】按钮。
*   **借阅发起表单弹窗**：要求填写：申请人、借阅天数（单选：1/3/7/15/30天）、详细申请理由。
*   **溯源回答与线索挖掘 (知识图谱)**：
    返回 AI 生成的总结性回复卡片，附带引文链接溯源。系统提供跨文档的时间轴查询、人物/机构关系连线图谱，支持多轮对话挖掘。

### 七、 借阅审批流转中心 (个人利用空间)
**1. 页面定位**：借阅生命周期控制台。
**2. 界面清单与功能细节**：
*   **我的借阅/收藏夹 (普通用户视窗)**：
    *   分 Tab 页签展示：【借阅申请中】、【借阅已通过】、【被驳回】、【历史浏览】、【我的收藏】。
    *   【借阅已通过】列表带有倒计时标识，**时间到期后防盗链彻底失效，无法打开**。
    *   **安全只读控制**：从此模块打开的图文详情页，系统**强制切入只读沙盒模式**，所有编辑修改、导出按钮隐藏，并满屏覆盖带当前操作人姓名的**防飞白水印**。
*   **三级协同审批台 (管理层工作流)**：
    *   流转链路：申请人提交 -> **申请人直属领导审批** -> **档案归属单位责任人审批** -> **系统档案管理员终审授权**。
    *   审批操作强制要求录入“审批意见”。驳回后流程终止并触发站内信通知。

### 八、 系统底层配置与编码安全管理
**1. 页面定位**：超级管理员系统基建控制台。
**2. 界面清单与功能细节**：
*   **【重点】档案动态编码规则引擎 (基于 DA/T 13-2022)**：
    超级管理员可在此配置各种档案门类的档号编制结构模板。
    *   支持规则：【单层文件】(如：全宗号-分类号-件号)、【文件-案卷组合】(如：全宗号-分类号-案卷号-卷内顺序号)。
    *   **实时级联更迭**：若系统运维中途修改了某类档案的编码拼接规则（如在中间加入了“年度”变量），系统后台启动静默批处理，**历史入库的所有相关档案的档号属性自动化完成翻新和同步级联更新**。
*   **强互斥机制的独立标签库**：
    创建多种维度的独立标签库集合。单库内最多且只能关联1个子标签记录（互斥唯一），但文件可横向贯穿多个库挂接复合标签。标签节点修改牵一发而动全身自动同步。
*   **数据隔离与向上同构授权**：
    *   普通用户查询范围锁定于本部门（支队）。
    *   职务层级隔离：高层级职务的用户，天然拥有平行穿透覆盖直系下级部门数据域的查询权限。
*   **大模型算力与基础字典维护**：配置 API 秘钥，维护数据字典（档案门类、保管期限枚举值等）。
